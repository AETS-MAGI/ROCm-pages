Tensile

このページで得られる理解：Tensile が生成した GEMM kernel は、どの世代に向けてビルドされているのか——ソースコードの登録状況と、実際に dispatch されるかどうかのギャップを記録する。

What you'll gain here: Which GPU generations Tensile-generated GEMM kernels are built for — and the gap between source registration and whether they're actually dispatched.

このページの結論Page Conclusion

  • gfx900（MI25）・gfx942（MI300X）・gfx1201（RX9070XT）の 3 世代はいずれも Tensile の BASE_ARCHITECTURES に登録済みで、ビルド対象として扱われる。
  • gfx1201 は逆アセンブリで Cijk_* family の実在を確認済み（wavefront_size=32、v_mfma なし）。
  • gfx900 は登録されているが、ggml-hip 側に gfx900 向け custom kernel が存在しないため、BLAS 経路での Tensile kernel 利用が実際の主経路である可能性が高い。
  • dispatch-safe observer なしには、catalog に含まれる kernel と実際に選ばれる kernel を結びつけることはできない。
  • All three generations — gfx900 (MI25), gfx942 (MI300X), gfx1201 (RX9070XT) — are registered in Tensile's BASE_ARCHITECTURES and treated as build targets.
  • gfx1201: Cijk_* family confirmed via disassembly (wavefront_size=32, no v_mfma).
  • gfx900 is registered, but since ggml-hip has no gfx900 custom kernels, Tensile kernels via the BLAS path are likely the primary route on MI25.
  • Without a dispatch-safe observer, linking catalog entries to actually-selected kernels is not possible.

ROCm スタックにおける位置づけPosition in the ROCm Stack

Tensile は rocBLAS（一次）と hipBLASLt（TensileLite 経由）のバックエンドカーネルジェネレータです。GPU アーキテクチャ仕様を入力に、ベンチマーク自動調整（auto-tuning）を経てアセンブリレベルの最適 kernel を出力し、hsaco として rocblas/library/ に配置します。

Tensile is the backend kernel generator for rocBLAS (primary) and hipBLASLt (via TensileLite). Given GPU architecture specifications, it produces assembly-level optimized kernels through automated benchmarking (auto-tuning), placing the resulting hsacos under rocblas/library/.

Tensile (ビルド時at build time) → Cijk_*_gfx{900,942,1201}.hsaco → rocblas/library/Kernels.so-000-gfx*.hsaco 実行時（rocBLAS 経由）:At runtime (via rocBLAS): rocBLAS → Processor enum lookup → Cijk_* kernel dispatch

gfx1201 実機観測（RX9070XT）gfx1201 Live Observation (RX9070XT)

観測ポイント：gfx1201 向け Tensile kernel が実際に bundle に存在し、逆アセンブリで確認できるか。

Observation point: Whether gfx1201 Tensile kernels actually exist in the live bundle and can be confirmed via disassembly.

  • RX9070XT 実機 bundle（/usr/local/lib/ollama/rocm/rocblas/library/）に gfx1201 ターゲットのファイルが 56 本存在（Phase C 確認）。
  • Kernels.so-000-gfx1201.hsaco（standalone ELF AMD GPU object）が含まれることを確認。
  • Phase E 逆アセンブリで Cijk_S_GA（sgpr:14, vgpr:10）・Cijk_S_PostGSU3（sgpr:21, vgpr:11）等を確認。いずれも wavefront_size=32。
  • v_mfma 命令は現行 inspect ウィンドウでは検出なし（matrix multiply-accumulate の未使用を示唆するが、inspect window が全 kernel をカバーしているわけではない）。
  • 56 gfx1201-targeted files found in the RX9070XT live bundle (/usr/local/lib/ollama/rocm/rocblas/library/) (Phase C confirmed).
  • Kernels.so-000-gfx1201.hsaco (standalone ELF AMD GPU object) confirmed present.
  • Phase E disassembly confirmed Cijk_S_GA (sgpr:14, vgpr:10) and Cijk_S_PostGSU3 (sgpr:21, vgpr:11), both wavefront_size=32.
  • No v_mfma instructions in the current inspection window (suggests matrix multiply-accumulate not used, but window does not cover all kernels).
