rocBLAS

AMD GPU 向けの BLAS 実装ライブラリです。HIP で実装された Level 1/2/3 BLAS 演算を提供し、Tensile が生成した per-target GEMM kernel を内包しています。LLM 推論では hipBLAS 経由または直接 BLAS fallback として呼び出されます。

AMD's BLAS implementation library for AMD GPUs. Provides Level 1/2/3 BLAS operations implemented in HIP, incorporating per-target GEMM kernels generated by Tensile. In LLM inference it is called via hipBLAS or as a direct BLAS fallback.

ROCm スタックにおける位置づけPosition in the ROCm Stack

rocBLAS は ROCm 数学スタックの基盤層です。hipBLAS・MIOpen・ユーザーアプリケーションから呼び出され、GEMM（行列積）を含む BLAS 演算を実行します。kernel は Tensile によって GPU アーキテクチャごとに事前生成され、rocblas/library/ 以下に配置されます。

rocBLAS is the foundation layer of the ROCm math stack. It is called by hipBLAS, MIOpen, and user applications to execute BLAS operations including GEMM. Kernels are pre-generated per GPU architecture by Tensile and placed under rocblas/library/.

観測済み事実（MI25 / RX9070XT）Observed Facts (MI25 / RX9070XT)

  • MI25 では ROCBLAS_TENSILE_LIBPATH 差分を用いた lane 比較対照実験を実施し、AETS lane / system lane 間の throughput 差を観測しました。
  • RX9070XT では ROCBLAS_LAYER=9 で rocblas_create_handle,atomics_allowed のみ記録。per-GEMM attribution には不足しています（Phase B 観測）。
  • rocBLAS の library/src/handle.cpp で rocblas_initialize() が gfx1201 の handle 初期化時に呼ばれることをソース確認しました。
  • For MI25, controlled lane comparison experiments using ROCBLAS_TENSILE_LIBPATH differences observed throughput differences between the AETS and system lanes.
  • For RX9070XT, ROCBLAS_LAYER=9 only recorded rocblas_create_handle,atomics_allowed — insufficient for per-GEMM attribution (Phase B observation).
  • Source-confirmed that rocblas_initialize() is called during gfx1201 handle initialization in library/src/handle.cpp.
